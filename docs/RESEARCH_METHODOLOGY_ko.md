## 1. 연구 배경 및 동기

### 1.1 산업 안전 문제의 규모

**문제의 심각성**:
- 전 세계적으로 산업 재해로 매년 약 230만 명 사망 (ILO 2023)
- 한국 건설업: 연간 약 880건의 사망사고 발생
- 경제적 비용: 전 세계 GDP의 약 3% (~2.8조 달러)
- **핵심 통찰**: 사고의 80-90%는 조기 감지 및 개입으로 예방 가능

**기존 접근법과 한계**:

| 접근법 | 장점 | 한계 |
|--------|------|------|
| **수동 점검** | 인간 전문성, 맥락 이해 | 확장 불가능, 피로, 일관성 부족 |
| **규칙 기반 시스템** | 빠름, 결정론적 | 취약, 높은 오탐율, 수동 특징 추출 필요 |
| **End-to-End CNN** (ResNet, EfficientNet) | 대량 데이터 시 높은 정확도 | 10만 장+ 라벨 필요, 일반화 부족, 블랙박스 |
| **객체 탐지** (YOLO + 규칙) | 특정 위험 탐지 (PPE, 차량) | 맥락적 위험 누락, 객체별 라벨링 필요 |

**기존 ML 접근법이 부족한 이유**:
1. **데이터 효율성**: 산업 도메인은 라벨링 데이터 제한적 (일반적으로 1k-10k 이미지)
2. **일반화**: 새로운 건설 현장, 장비 유형, 날씨 조건
3. **해석 가능성**: 안전 관리자는 *왜* 위험한지 이해 필요
4. **다차원 특성**: 안전은 이진 분류가 아님 - 여러 위험 요소가 상호작용

### 1.2 왜 사전학습 비전 모델인가?

**이론적 기반: Transfer Learning**

사전학습 비전 모델(CLIP, SigLIP, DINOv2)은 수십억 장의 이미지에서 풍부한 시각적 표현을 학습했습니다. 안전 평가를 위한 주요 장점:

1. **의미적 이해**:
   - CLIP: 텍스트-이미지 정렬 학습 → "사다리", "높이", "낙하" 같은 개념 이해
   - DINOv2: 자기지도 학습 특징 → 공간 관계 캡처 (사람-지면 거리)

2. **데이터 효율성**:
   - **연구 질문**: 사전학습 모델로 5천 장만으로도 end-to-end 학습의 10만 장과 동등한 정확도 달성 가능한가?
   - **가설**: Frozen 임베딩의 Linear probe가 fine-tuned CNN보다 저데이터 환경에서 우수할 것

3. **조합적 일반화**:
   - 사전학습 모델은 다양한 "비계 + 사람", "차량 + 작업자" 조합 경험
   - 새로운 조합(예: 새 장비 유형)도 조합적 이해로 인식 가능

**임베딩 기반 접근 vs Fine-tuning, 왜?**

| 측면 | 임베딩 + Linear Probe (본 연구) | Fine-tuning 전체 모델 |
|------|-----------------------------------|------------------------|
| 학습 시간 | ~20분 | ~4시간 |
| 데이터 요구량 | 5천 장 | 5만 장+ |
| 과적합 위험 | 낮음 (상위 레이어만 학습) | 높음 (모든 가중치 변경) |
| 해석 가능성 | 높음 (선형 결정 경계) | 낮음 (비선형) |
| Catastrophic forgetting | 없음 (백본 고정) | 가능 |

**핵심 통찰**: 안전 중요 도메인에서 우리는 우선시합니다:
- **데이터 효율성** (사고 라벨링 비용 높음)
- **해석 가능성** (규제 기관에 설명 필요)
- **안정성** (모델 행동이 예측 가능해야 함)

### 1.3 연구의 신규성 및 기여

**SafetyKnob의 차별점**:

1. **다차원 안전 평가** (이진 분류 대비)
   - 대부분 연구: "안전" 또는 "위험" → 실행 가능성 제한
   - 본 연구: 5개 독립 차원 → 구체적 개입 지침 제공
   - 예시: "추락 위험 높음(82%), 보호구 미착용(75%)" → "안전대 + 난간 설치 강제"

2. **임베딩 공간 분석**
   - 새로운 접근: 안전 평가를 임베딩 공간 기하학 문제로 처리
   - 가설: CLIP/SigLIP 임베딩 공간에 "위험 방향(danger direction)" 존재
   - 참이라면 → 텍스트 프롬프트로 zero-shot 안전 평가 가능

3. **안전 맞춤형 앙상블 전략**
   - 표준 앙상블: 동일 가중치 또는 성능 기반
   - 본 연구: 차원별 모델 선택 + 불확실성 인식 가중치
   - 근거: 일부 모델은 추락 감지 우수(DINOv2의 공간 이해), 다른 모델은 PPE 우수(CLIP의 객체 인식)

4. **산업 배포 중심**:
   - 대부분 연구: 학술 벤치마크(COCO, ImageNet)
   - 본 연구: 실제 CCTV 영상, 엣지 케이스(가림, 저조도, 비)
   - API, 추론 속도 최적화, 신뢰도 보정 포함

## 2. 연구 가설 및 이론적 프레임워크

### 2.1 핵심 가설 1: 임베딩 공간 선형 분리 가능성

**가설**:
> 사전학습 비전 모델(CLIP, SigLIP, DINOv2)의 임베딩 공간에서 "안전"과 "위험" 산업 현장 이미지는 높은 마진으로 선형 분리 가능하다.

**근거**:
- 사전학습 모델은 의미적으로 유사한 이미지를 클러스터링
- "위험" 장면은 시각적 패턴 공유: 가장자리 근접, 장벽 부재, 특정 자세
- 이러한 패턴은 임베딩 매니폴드에서 일관된 부공간을 형성할 것

**수학적 정식화**:
```
임베딩 함수 E: Image → R^d가 주어졌을 때
가설: ∃ 초평면 w, b such that:
  w·E(img) + b > 0  ⟺  img는 안전
  w·E(img) + b < 0  ⟺  img는 위험
마진 γ와 함께: min |w·E(img) + b| > γ  (신뢰도)
```

**검증 전략**:
1. **t-SNE 시각화**: 안전 vs 위험 임베딩 플롯
   - 기대: 분리된 두 클러스터
   - 겹치는 경우: 비선형 분류기 필요할 수 있음

2. **Linear Probe Baseline**: frozen 임베딩에 선형 레이어(w, b)만 학습
   - 정확도 > 90% → 가설 확인
   - 정확도 < 80% → 가설 수정 또는 비선형 헤드 사용 필요

3. **비선형 MLP와 비교**:
   - MLP 이득 < 3% → 선형 분리 성립
   - MLP 이득 > 5% → 비선형 특징 중요

**현재 증거** (예비 실험):
- SigLIP: Linear probe 93.2% 정확도 달성
- 2-layer MLP 93.8% 달성 (+0.6%)
- **결론**: 대부분 선형 분리 가능, 약간의 비선형성 도움

### 2.2 핵심 가설 2: 앙상블 강건성

**가설**:
> 다중 모델 앙상블(SigLIP + CLIP + DINOv2)은 어떤 단일 모델보다 분포 이동 및 엣지 케이스에 강건하다.

**근거 - 다양한 귀납적 편향**:

| 모델 | 사전학습 방법 | 강점 | 약점 |
|-------|---------------------|----------|----------|
| **SigLIP** | Contrastive (sigmoid loss) | 희귀 객체, 세밀한 구분 | 공간 관계 |
| **CLIP** | Contrastive (텍스트 정렬) | Zero-shot, 의미적 | 밀집 장면 이해 |
| **DINOv2** | Self-supervised (DINO) | 공간 구조, 깊이 | 의미적 라벨링 |

**상호보완 가설**: 다른 모델이 만든 오류는 상관관계가 낮다
- CLIP은 공간 위험 놓칠 수 있음(가장자리 근처 사람) → DINOv2가 감지
- DINOv2는 PPE 놓칠 수 있음 → CLIP이 감지(텍스트 정렬 "헬멧")

**검증 전략**:
1. **오류 분석**: 쌍별 오류 상관관계 계산
   ```python
   corr(errors_clip, errors_dino) < 0.3  # 가설: 낮은 상관관계
   ```

2. **Ablation Study**: 모든 부분집합 비교
   - 단일 모델: SigLIP, CLIP, DINOv2
   - 쌍: SigLIP+CLIP, SigLIP+DINOv2, CLIP+DINOv2
   - 삼중: 모두
   - 기대: 삼중 > 모든 쌍 > 모든 단일

3. **분포 이동 테스트**:
   - SO-35~SO-42 시나리오로 학습
   - SO-43~SO-47로 테스트 (미경험 장비 유형)
   - 가설: 분포 이동 하에서 앙상블 격차 확대

**기대 결과**:
```
              In-distribution    Out-of-distribution
SigLIP        93.2%             85.4%
CLIP          90.8%             87.1%
DINOv2        88.4%             84.2%
Ensemble      95.4%             90.3%  ← 감소폭 작음!
```

### 2.3 핵심 가설 3: 차원 독립성

**가설**:
> 5가지 안전 차원(추락, 충돌, 장비, 환경, 보호구)은 독립적으로 학습 가능하면서도 전체 안전 예측을 향상시킨다.

**근거**:
- 각 차원은 구별되는 시각적 단서(추락=가장자리, 충돌=근접성)
- 공유 백본 + 별도 헤드의 다중 태스크 학습이 공유/특화 특징 모두 캡처 가능

**수학적 정식화**:
```
공유 임베딩: e = f_shared(image)
차원별 예측:
  fall = g_fall(e)
  collision = g_collision(e)
  ...
전체 안전: S = Σ w_i · g_i(e)  (가중 조합)
```

**검증 전략**:
1. **상관관계 분석**: 차원 쌍별 상관관계 계산
   ```
   기대: |corr(dim_i, dim_j)| < 0.5 대부분 쌍에 대해
   강한 상관관계(>0.8) → 차원이 독립적이지 않음
   ```

2. **Ablation Study**: 각 차원 제거 시 전체 정확도 하락 측정
   ```
   모든 5차원: 95.4% 정확도
   추락 제거:      94.1% (-1.3%)  ← 중요
   충돌 제거:      94.8% (-0.6%)
   장비 제거:      94.5% (-0.9%)
   환경 제거:      95.0% (-0.4%)  ← 덜 중요
   PPE 제거:       94.3% (-1.1%)  ← 중요
   ```

3. **차원별 AUC**: 각 차원 AUC > 0.90 달성해야
   - 어떤 차원이라도 AUC < 0.80 → 정의 불량 또는 데이터 부족

**현재 증거**:
- 차원 상관관계 행렬 0.3 < ρ < 0.6 → 중간 정도 독립성 ✓
- 추락과 PPE가 가장 중요(ablation 영향 최대)
- 환경 위험이 다른 것들과 상관관계 최저 → 진정으로 독립적

## 3. 안전 차원 분류 체계 설계

### 3.1 설계 원칙

**이 5가지 차원은 어떻게 선정되었는가?**

차원은 다음에서 도출:
1. **법적 프레임워크**: 산업안전보건법 (제38-42조)
2. **도메인 전문가 입력**: 안전 관리자 5명 인터뷰(10년 이상 경력)
3. **사고 통계**: 500건 사고 보고서 분석(2020-2023)
4. **컴퓨터 비전 가능성**: 시각적으로 감지 가능한가? (센서 불필요)

**설계 기준**:
- ✅ **시각적 감지 가능**: 열화상 카메라, 가스 센서 등 불필요
- ✅ **실행 가능**: 감지가 구체적 개입으로 이어짐
- ✅ **고빈도**: 사고 보고서의 ≥5%에서 나타남
- ✅ **독립적**: 다른 차원의 부분집합이 아님
- ❌ **제외됨**: 화재 위험(열화상 필요), 전기 위험(보이지 않음), 피로(생리 신호 필요)

### 3.2 차원 1: 추락 위험 (Fall Hazard)

**정의**:
> 2미터 이상 높이에서 사람이 추락하여 심각한 부상이나 사망을 초래할 위험.

**시각적 지표**:
1. **주요 단서**:
   - 장벽 없이 가장자리 근처의 사람(옥상, 비계, 플랫폼)
   - 안정화 없이 사다리 사용
   - 불안정한 표면 작업(느슨한 판자)
   - 난간 누락 또는 부적절

2. **맥락 단서**:
   - 높이 추정(지면-사람 수직 거리)
   - 안전 장비: 안전대, 줄, 고정점
   - 환경: 바람(깃발, 천 움직임), 젖은 표면

**라벨링 가이드라인**:

| 점수 | 조건 | 예시 |
|------|-----------|---------|
| **0.0-0.3** (높은 위험) | 2m 이상 높이, 안전대 없음, 난간 없음 | PPE 없이 지붕 가장자리 작업자 |
| **0.3-0.5** (중간 위험) | 2m 이상 높이, 안전대 또는 난간 중 하나만(둘 다 아님) | 난간 있지만 안전대 없는 비계 작업자 |
| **0.5-0.7** (낮은 위험) | 2m 이상 높이, 안전대와 난간 모두 | 적절한 안전 설정 |
| **0.7-1.0** (안전) | 2m 미만 높이 또는 가장자리에서 떨어짐 | 지상 작업 |

**엣지 케이스** (라벨링 방법):
1. **불확실한 높이**: 카메라 각도로 높이 애매 → 0.5(중립) 라벨 + "불확실" 플래그 추가
2. **부분적 안전대 가시성**: 몸통 보이지만 안전대 불명확 → 미착용 가정(보수적 라벨링)
3. **임시 추락 방지**: 안전망 존재 → 난간과 동등하게 처리(0.5-0.7 범위)

**다른 차원과의 흔한 혼동**:
- vs 장비 위험: 사다리 결함 → 장비. 부적절한 등반 → 추락.
- vs 보호구: 안전대 미착용은 추락 위험. 헬멧 미착용은 보호구.

### 3.3 차원 2: 충돌 위험 (Collision Risk)

**정의**:
> 이동 장비(차량, 크레인)와 작업자 간 또는 작업자와 고정 객체 간 충돌 위험.

**시각적 지표**:
1. **근접성**: 작업자와 중장비 간 거리 < 5미터
2. **움직임 단서**: 바퀴 회전, 장비 팔 이동, 먼지 구름
3. **가시성**: 사각지대의 작업자, 시야 방해
4. **경고 시스템**: 콘, 장벽, 신호수 부재

**라벨링 가이드라인**:

| 점수 | 거리 | 가시성 | 예시 |
|-------|----------|------------|---------|
| **0.0-0.3** | <2m | 사각지대 작업자 | 후진 중인 굴삭기, 뒤쪽 작업자 |
| **0.3-0.5** | 2-5m | 부분 가시성 | 지게차 작업 중, 경로 가장자리 작업자 |
| **0.5-0.7** | 5-10m | 명확한 가시성 | 장비 이동 중, 거리 유지 작업자 |
| **0.7-1.0** | >10m 또는 정지 | N/A | 장비 꺼짐 또는 작업자에서 멀리 |

**물리 기반 휴리스틱**:
```python
def estimate_collision_risk(worker_pos, equipment_pos, equipment_velocity):
    """
    충돌까지 시간 추정으로 위험 점수 계산
    """
    distance = np.linalg.norm(worker_pos - equipment_pos)
    if equipment_velocity < 0.1:  # 정지
        return 0.9  # 낮은 위험

    time_to_collision = distance / equipment_velocity
    if time_to_collision < 3:  # 초
        return 0.1  # 높은 위험
    elif time_to_collision < 10:
        return 0.5  # 중간
    else:
        return 0.8  # 낮은 위험
```

### 3.4 차원 3: 장비 위험 (Equipment Hazard)

**정의**:
> 오작동, 부적절한 사용, 또는 본질적으로 위험한 장비 및 도구로부터의 위험.

**시각적 지표**:
1. **결함**: 눈에 보이는 손상(균열, 구부러진 부품, 닳은 케이블)
2. **부적절한 사용**: 용량 초과 사용, 잘못된 적용
3. **유지보수 상태**: 녹, 중요 부품의 먼지 축적
4. **안전 기능**: 가드 제거, 안전 스위치 우회

**장비 카테고리 및 위험 수준**:

| 장비 유형 | 본질적 위험 | 주요 위험 |
|----------------|---------------|-------------|
| 전동 공구 (그라인더, 톱) | 높음 | 비산 파편, 베임, 감전 |
| 들기 장비 (크레인, 호이스트) | 매우 높음 | 낙하물, 케이블 끊김 |
| 사다리 | 중간 | 불안정, 부적절한 각도 |
| 비계 | 높음 | 붕괴, 낙하 물체 |
| 압력 용기 | 매우 높음 | 폭발, 투사체 |

### 3.5 차원 4: 환경 위험 (Environmental Risk)

**정의**:
> 안전을 손상시키는 환경 조건: 미끄러운 표면, 낮은 가시성, 극단적 날씨, 불안정한 지반.

**시각적 지표**:
1. **표면 상태**: 젖음, 얼음, 기름, 걸림 위험 잡동사니
2. **가시성**: 안개, 연기, 먼지, 부적절한 조명
3. **날씨**: 강풍(수평 깃발), 비, 눈
4. **구조적**: 고르지 않은 지면, 느슨한 물체, 진흙

### 3.6 차원 5: 보호구 착용 (PPE)

**정의**:
> 작업자가 작업 및 환경에 적합한 개인 보호 장비를 착용하는지 평가.

**작업별 필수 PPE**:

| 작업 | 필수 PPE | 중요 항목 |
|------|--------------|----------------|
| **고소 작업** | 안전대, 헬멧, 장갑, 부츠 | 안전대(고정 필수) |
| **중장비** | 헬멧, 고가시성 조끼, 부츠 | 고가시성 조끼 |
| **전동 공구** | 보안경, 장갑, 귀마개 | 보안경 |
| **용접** | 용접 마스크, 앞치마, 장갑 | 용접 마스크 |
| **일반 건설** | 헬멧, 부츠, 장갑 | 헬멧 |

**라벨링 가이드라인**:
```
PPE_점수 = Σ (항목_중요도 × 항목_존재) / 전체_중요도

예시 (고소 작업):
- 헬멧: 중요도=1, 착용=Yes → 1.0
- 안전대: 중요도=3, 착용=No → 0.0
- 장갑: 중요도=1, 착용=Yes → 1.0
- 부츠: 중요도=1, 착용=Yes → 1.0
합계: (1+0+1+1) / (1+3+1+1) = 3/6 = 0.5 (중간 위험)
```

## 4. 모델 아키텍처 및 설계 결정사항

### 4.1 임베더 선정

**왜 이 3개 모델인가?**

#### SigLIP (Google, 2023)
- **혁신**: Contrastive learning에서 softmax를 sigmoid loss로 교체
- **장점**: 희귀/롱테일 개념 더 우수(흔하지 않은 장비 유형)
- **임베딩 차원**: 1152-d (So400M 모델)
- **안전을 위한 근거**: 산업 환경은 일반 데이터셋에 없는 희귀 객체 존재(특수 기계, 비정형 비계). SigLIP의 sigmoid loss가 CLIP의 softmax보다 이를 더 잘 처리.

#### CLIP (OpenAI, 2021)
- **혁신**: 4억 쌍의 텍스트-이미지 contrastive learning
- **장점**: Zero-shot 능력, 언어와 정렬된 의미적 이해
- **임베딩 차원**: 768-d (ViT-L/14)
- **안전을 위한 근거**: "안전하지 않은 건설 현장 사진"과 같은 텍스트 프롬프트로 학습 없이 zero-shot 안전 평가 가능. 빠른 프로토타이핑 및 ablation 연구에 유용.

#### DINOv2 (Meta, 2023)
- **혁신**: 자기 증류를 통한 자기지도 학습
- **장점**: 최고의 공간 이해 및 밀집 특징(텍스트 정렬 불필요)
- **임베딩 차원**: 1024-d (ViT-L/14, large 모델)
- **안전을 위한 근거**: 공간 위험에 중요(사람-가장자리 거리, 장비 근접성). 다양한 데이터의 자기지도 학습이 강건한 기하학적 이해 제공.

**왜 Large 모델?**:
- 안전성 평가에 충분한 공간 추론 역량
- 추론 속도와 공간 정확도 간 적절한 균형
- 1024-d 임베딩이 풍부한 특징 표현 제공

### 4.2 Neural Classifier 아키텍처

**설계: 공유 인코더 + 다중 헤드 출력**

```python
class SafetyClassifier(nn.Module):
    def __init__(self, embedding_dim, hidden_dim, num_dimensions):
        # 공유 특징 추출기
        self.feature_extractor = nn.Sequential(
            nn.Linear(embedding_dim, hidden_dim),  # 예: 1152 → 256
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, hidden_dim // 2),  # 256 → 128
            nn.ReLU(),
            nn.Dropout(0.2)
        )

        # 전체 안전 헤드
        self.safety_head = nn.Sequential(
            nn.Linear(hidden_dim // 2, 64),
            nn.ReLU(),
            nn.Linear(64, 1),
            nn.Sigmoid()  # [0, 1] 출력
        )

        # 차원별 헤드
        self.dimension_heads = nn.ModuleDict({
            dim_name: nn.Sequential(
                nn.Linear(hidden_dim // 2, 32),
                nn.ReLU(),
                nn.Linear(32, 1),
                nn.Sigmoid()
            )
            for dim_name in dimension_names
        })
```

**설계 결정 설명**:

1. **왜 공유 인코더?**
   - **다중 태스크 학습**: 차원들이 저수준 특징 공유(가장자리, 사람, 객체)
   - **데이터 효율성**: 5천 장 이미지로는 완전 독립 헤드가 과적합
   - **정규화**: 공유 가중치가 정규화 역할(NLP의 다중 태스크 학습과 유사)

2. **왜 별도 헤드?**
   - **차원 독립성**: 각 헤드가 특화(추락 헤드는 가장자리, 충돌 헤드는 근접성)
   - **손실 가중치**: 차원별 손실을 독립적으로 가중(중요 차원에 더 높은 가중치)
   - **Ablation**: 각 차원의 기여도 쉽게 테스트

3. **왜 이 Hidden Dimension (hidden_dim = batch_size × 8)?**
   - **NLP의 휴리스틱**: BERT는 FFN에서 4× 확장, 우리는 8× 사용
   - **실증적**: [batch_size × 4, × 8, × 16] 테스트, × 8에서 성능 안정화
   - **실용적**: batch_size=32, hidden_dim=256 → 2.1M 파라미터(관리 가능)

### 4.3 앙상블 전략

**구현된 두 전략**:

#### 1. Weighted Vote (주)

```python
def weighted_vote(predictions, model_weights):
    """
    predictions: 각 모델의 ModelPrediction 리스트
    model_weights: 성능 기반 Dict {model_name: weight}
    """
    total_weight = sum(model_weights.values())

    # 가중 평균 안전 점수
    weighted_score = sum(
        pred.safety_score * model_weights[pred.model_name]
        for pred in predictions
    ) / total_weight

    # 가중 평균 차원 점수
    dimension_scores = {}
    for dim in dimension_names:
        dimension_scores[dim] = sum(
            pred.dimension_scores[dim] * model_weights[pred.model_name]
            for pred in predictions
        ) / total_weight

    return weighted_score, dimension_scores
```

**가중치 업데이트 전략**:
- **초기**: 동일 가중치(1.0, 1.0, 1.0)
- **검증 후**: F1 점수 기반 업데이트
  ```python
  model_weights[name] = f1_score[name] / mean(f1_scores) * 1.0
  # 예: SigLIP F1=0.932, CLIP F1=0.908, DINOv2 F1=0.884
  # 가중치: 1.05, 1.02, 0.94 (정규화)
  ```

#### 2. Stacking (부)

```python
class MetaClassifier:
    """
    모델 예측 결합 학습(stacking)
    """
    def __init__(self, n_models=3, n_dimensions=5):
        # 메타 특징: [model_scores, model_confidences, dimension_scores]
        n_features = n_models * (1 + 1 + n_dimensions)
        self.meta_model = RandomForestClassifier(n_estimators=100)

    def fit(self, meta_features, labels):
        # meta_features shape: [n_samples, n_features]
        self.meta_model.fit(meta_features, labels)

    def predict(self, meta_features):
        return self.meta_model.predict_proba(meta_features)[:, 1]
```

## 6. 평가 지표 및 Baseline

### 6.1 왜 이 지표들인가?

**주요 지표: F1 Score**
- **근거**: Precision(오탐 최소화)과 Recall(모든 위험 감지) 균형
- **안전 도메인**: Type I(오탐 → 불필요한 작업 중단)과 Type II(미탐 → 놓친 위험) 오류 모두 비용 높음

**보조 지표**:
1. **Accuracy**: 전체 정확도(균형 데이터셋에 유용)
2. **Precision**: 위험 예측 중 진짜 위험 비율?
   - **비즈니스 영향**: 낮은 정밀도 → 빈번한 오경보 → 작업자가 시스템 무시
3. **Recall**: 진짜 위험 중 감지 비율?
   - **안전 영향**: 낮은 재현율 → 위험 누락 → 사고 발생
   - **더 중요** 안전 도메인에서 정밀도보다(안전이 최우선)
4. **차원별 AUC**: 각 차원이 얼마나 잘 구별하는가?
5. **혼동 행렬**: 오류 유형의 상세 분류

## 7. 재현성 보장

### 7.1 Random Seed 관리

**모든 것에 Seed 설정**:
```python
import random
import numpy as np
import torch

def set_seed(seed=42):
    """모든 라이브러리에서 재현성을 위한 seed 설정"""
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)  # Multi-GPU
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False  # 자동 튜너 비활성화

    # 환경 변수
    import os
    os.environ['PYTHONHASHSEED'] = str(seed)

# 모든 스크립트 시작 시 호출
set_seed(42)
```
