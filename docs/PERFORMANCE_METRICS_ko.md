## 왜 이 지표들이 중요한가?

### 산업 재해의 현실

**국제 노동 기구(ILO) 통계 (2023)**:
- 전 세계 연간 산업 재해 사망자: **약 230만 명**
- 비치명적 사고: **연간 3억 건 이상**
- 경제적 손실: **전 세계 GDP의 약 4% (약 3조 달러)**

**국내 건설업 재해 현황 (고용노동부, 2023)**:
- 건설업 사망 재해: **연간 400~500명**
- 전체 산업 재해 사망의 **약 50%가 건설업**
- 추락 사고: 건설업 사망 재해의 **50% 이상**
- 재해 1건당 평균 비용: **약 3억 5천만 원**
  - 직접 비용 (치료비, 보상금): 1억 원
  - 간접 비용 (작업 중단, 생산성 손실): 2억 5천만 원

### AI 안전 시스템의 필요성

**기존 방식의 한계**:
```
인력 기반 모니터링:
- 안전 관리자 1명당 모니터링 가능 현장: 5~10개
- 실시간 모니터링 불가능
- 주관적 판단에 의존
- 피로도 누적 시 탐지율 저하

결과:
- 위험 상황 발견율: 40~60%
- 사고 후 대응 중심
```

**AI 시스템 도입 효과 (예상)**:
```
SafetyKnob 적용 시:
- 모니터링 현장: 무제한 (카메라 기준)
- 실시간 24시간 감시 가능
- 일관된 기준 적용
- 피로도 없음

기대 효과:
- 위험 상황 발견율: 85~95%
- 사고 발생률 감소: 30~50%
- 연간 절감 비용 (1,000개 현장 기준): 약 100~200억 원
```

### 성능 지표가 실제 생명을 좌우한다

**F1 Score 0.95 vs 0.85의 차이**:
```
시나리오: 하루 1,000개 현장, 각 현장 10개 위험 상황

F1 0.95 시스템:
  - 탐지 성공: 9,500개
  - 놓친 위험: 500개
  - 예상 사고: 5~10건 (치명률 1~2%)

F1 0.85 시스템:
  - 탐지 성공: 8,500개
  - 놓친 위험: 1,500개
  - 예상 사고: 15~30건 (치명률 1~2%)

차이:
  - 추가 예방 사고: 10~20건/일
  - 연간 예방 효과: 3,650~7,300건
  - 절감 비용: 약 1조 원
```

이것이 우리가 **F1 Score 1%를 위해 노력하는 이유**입니다.

## 1. Val F1 Score (Validation F1)

### 정의
**F1 Score**는 정밀도(Precision)와 재현율(Recall)의 조화 평균으로, 불균형한 데이터셋에서 모델의 실제 성능을 가장 잘 나타내는 지표입니다.

### 계산 방법
```
F1 = 2 × (Precision × Recall) / (Precision + Recall)

where:
  Precision = TP / (TP + FP)  # 위험하다고 예측한 것 중 실제 위험의 비율
  Recall = TP / (TP + FN)     # 실제 위험 중 올바르게 탐지한 비율
```

### 구성 요소

**True Positive (TP)**: 위험한 상황을 위험하다고 정확히 판단
```
이미지: 안전모 미착용 작업자
예측: 위험 ✅
실제: 위험
→ TP
```

**False Positive (FP)**: 안전한 상황을 위험하다고 잘못 판단
```
이미지: 안전 장비 완벽 착용
예측: 위험 ❌
실제: 안전
→ FP (오경보)
```

**False Negative (FN)**: 위험한 상황을 안전하다고 잘못 판단
```
이미지: 고소 작업 중 안전벨트 미착용
예측: 안전 ❌
실제: 위험
→ FN (위험 누락 - 가장 치명적!)
```

**True Negative (TN)**: 안전한 상황을 안전하다고 정확히 판단
```
이미지: 안전 교육 중
예측: 안전 ✅
실제: 안전
→ TN
```

### 실제 의미

**현재 모델 성능 (2025년 10월 기준)**:

| 모델 | Val F1 | 100장당 오분류 | 의미 |
|------|--------|----------------|------|
| **SigLIP** | 0.9566 | 약 4~5장 | 거의 완벽한 수준 |
| **CLIP** | 0.8776 | 약 12장 | 실무 적용 가능 |
| **DINOv2** | 0.8408 | 약 16장 | 보조 시스템 가능 |

### 산업 안전 관점에서의 중요성

**False Negative (위험 누락)의 심각성**:
```
FN이 1%만 존재해도:
- 매일 100개 현장 모니터링
- 1개 위험 상황 놓침
- → 잠재적 사고 발생 가능
```

**False Positive (오경보)의 영향**:
```
FP가 10%:
- 실제 안전한 상황에서 경보
- 작업 중단 및 생산성 저하
- 경보 피로도(alarm fatigue) 발생
```

### 최적 균형점
```
산업 안전 시스템의 이상적 F1 Score:
- 0.95 이상: 실무 독립 운영 가능
- 0.90~0.95: 전문가 검토와 병행
- 0.85~0.90: 보조 도구로 활용
- 0.85 미만: 추가 개선 필요
```

## 2. Val AUC (Area Under ROC Curve)

### 정의
**AUC (Area Under the Curve)**는 ROC 곡선 아래 면적으로, 임계값(threshold)에 무관하게 분류기의 본질적인 성능을 측정합니다.

### ROC 곡선이란?
```
ROC Curve = True Positive Rate (TPR) vs False Positive Rate (FPR)

TPR (Sensitivity) = TP / (TP + FN)  # 실제 위험 중 탐지 비율
FPR (1 - Specificity) = FP / (FP + TN)  # 안전 중 잘못 탐지 비율
```

### AUC 값의 의미

| AUC 범위 | 분류 성능 | 설명 |
|----------|-----------|------|
| **1.0** | 완벽 | 모든 위험을 100% 정확히 분류 |
| **0.9~1.0** | 탁월 | 실무 적용 가능, 매우 높은 신뢰도 |
| **0.8~0.9** | 우수 | 실용적 수준 |
| **0.7~0.8** | 양호 | 개선 여지 있음 |
| **0.5~0.7** | 미흡 | 추가 학습 필요 |
| **0.5** | 무작위 | 동전 던지기와 동일 |

### 임계값 조정의 실제 사례

**보수적 설정 (Threshold = 0.3)**:
```
목표: 위험을 절대 놓치지 않음
결과:
  - TPR ↑ (위험 탐지율 높음)
  - FPR ↑ (오경보 많음)

사용 사례: 고위험 작업 (고소 작업, 화학 물질)
```

**균형 설정 (Threshold = 0.5)**:
```
목표: TPR과 FPR의 균형
결과:
  - TPR ↔ (적정 탐지율)
  - FPR ↔ (적정 오경보율)

사용 사례: 일반 건설 현장
```

**엄격한 설정 (Threshold = 0.7)**:
```
목표: 확실한 위험만 경보
결과:
  - TPR ↓ (위험 놓칠 수 있음)
  - FPR ↓ (오경보 적음)

사용 사례: 저위험 작업 (사무실, 경미한 작업)
```

### 현재 모델 성능

| 모델 | Val AUC | 평가 | 임계값 조정 여지 |
|------|---------|------|------------------|
| **SigLIP** | 0.9917 | 거의 완벽 | 매우 유연함 |
| **CLIP** | 0.9484 | 탁월 | 유연함 |
| **DINOv2** | 0.9229 | 탁월 | 적절함 |

**해석**:
- 모든 모델이 **0.92 이상**으로 실무 적용 가능
- SigLIP은 거의 완벽한 분류 능력 (AUC 99.2%)
- 임계값을 조정하여 다양한 상황에 최적화 가능

## 3. Avg Dim F1 (Average Dimension F1)

### 정의
5가지 안전 차원 각각에 대한 F1 Score의 평균값입니다.

### 5가지 안전 차원

#### 1. **Fall Hazard (낙상 위험)**
```
탐지 대상:
- 높은 곳에서의 작업 (비계, 사다리, 지붕)
- 안전 난간 부재
- 안전벨트 미착용
- 불안정한 발판

예시:
  이미지: 3층 비계에서 안전벨트 없이 작업
  Fall Hazard Score: 0.95 (매우 위험)
```

#### 2. **Collision Risk (충돌 위험)**
```
탐지 대상:
- 작업자와 중장비의 근접성
- 움직이는 차량 주변 작업
- 좁은 공간에서의 다중 작업
- 시야 확보 미흡

예시:
  이미지: 굴삭기 작동 중 후방에서 작업
  Collision Risk Score: 0.88 (위험)
```

#### 3. **Equipment Hazard (장비 위험)**
```
탐지 대상:
- 위험한 기계 조작 (톱, 드릴, 절단기)
- 전기 장비 노출
- 날카로운 도구 사용
- 기계 안전 장치 미작동

예시:
  이미지: 안전 커버 제거된 원형 톱 사용
  Equipment Hazard Score: 0.92 (매우 위험)
```

#### 4. **Environmental Risk (환경 위험)**
```
탐지 대상:
- 미끄러운 바닥 (물, 기름)
- 먼지나 연기로 인한 시야 방해
- 극한 날씨 (폭우, 강풍)
- 화재 또는 누출 위험

예시:
  이미지: 기름 유출된 바닥에서 작업
  Environmental Risk Score: 0.76 (주의)
```

#### 5. **Protective Gear (보호 장비)**
```
탐지 대상:
- 안전모 미착용
- 안전화 미착용
- 보안경/마스크 미착용
- 반사 조끼 미착용

예시:
  이미지: 안전모 없이 건설 현장 출입
  Protective Gear Score: 0.98 (보호구 미착용 확실)
```

### 왜 Avg Dim F1이 낮은가?

**전체 안전/위험 판단 (F1 95%)**:
```
Task: 이미지가 안전한가, 위험한가? (2-class)
난이도: ⭐⭐
결과: 매우 정확 (F1 0.95)
```

**5가지 차원 동시 판단 (Avg F1 53%)**:
```
Task: 5가지 위험 요인이 각각 존재하는가? (multi-label)
난이도: ⭐⭐⭐⭐⭐
결과: 어려움 (Avg F1 0.53)
```

**왜 어려운가?**:

1. **다중 레이블 문제**:
```
하나의 이미지에 여러 위험 요소 공존:
  - Fall Hazard: Yes
  - Collision Risk: Yes
  - Equipment Hazard: No
  - Environmental Risk: Yes
  - Protective Gear: Yes

→ 5개 독립 판단 모두 정확해야 함
```

2. **라벨링의 주관성**:
```
같은 이미지를 보고도:
  전문가 A: "Collision Risk 있음"
  전문가 B: "Collision Risk 없음 (거리 충분)"

→ Ground truth 자체가 불명확
```

3. **클래스 불균형**:
```
데이터셋 분포:
  Protective Gear: 8,000장 (많음) → F1 0.85
  Environmental Risk: 1,200장 (적음) → F1 0.36

→ 적은 데이터로 학습 어려움
```

### 차원별 성능 (SigLIP 모델 기준)

| 차원 | F1 Score | 난이도 | 개선 방향 |
|------|----------|--------|-----------|
| **Protective Gear** | 0.8475 | ⭐⭐ | 시각적으로 명확 |
| **Equipment Hazard** | 0.5383 | ⭐⭐⭐ | 장비 종류 다양함 |
| **Collision Risk** | 0.4454 | ⭐⭐⭐⭐ | 거리 판단 어려움 |
| **Fall Hazard** | 0.4096 | ⭐⭐⭐⭐ | 높이 인식 필요 |
| **Environmental Risk** | 0.3552 | ⭐⭐⭐⭐⭐ | 가장 주관적 |

**평균**: 0.5192

### 실무 활용 방법

**전체 판단 (F1 95%) 활용**:
```
1차 스크리닝:
  - 위험/안전 빠른 분류
  - 위험 이미지만 추출
  - 자동 경보 발송
```

**차원별 판단 (Avg F1 53%) 활용**:
```
2차 상세 분석:
  - 위험 유형 파악
  - 맞춤형 안전 조치 권고
  - 통계 분석 (어떤 위험이 많은가?)
```

## 4. 실무 적용 기준

### 안전 관리 시스템 요구 사항

**Level 1: 자동 경보 시스템**
```
요구 성능:
  - Val F1 ≥ 0.95
  - Val AUC ≥ 0.98
  - False Negative Rate < 5%

현재 적용 가능 모델: SigLIP ✅

사용 사례:
  - 고위험 구역 실시간 모니터링
  - 자동 경보 발송
  - 관리자 알림
```

**Level 2: 전문가 보조 도구**
```
요구 성능:
  - Val F1 ≥ 0.85
  - Val AUC ≥ 0.90
  - False Negative Rate < 15%

현재 적용 가능 모델: SigLIP, CLIP ✅

사용 사례:
  - 안전 관리자 의사결정 지원
  - 정기 안전 점검 보고서
  - 위험 구역 우선순위 지정
```

**Level 3: 통계 분석 도구**
```
요구 성능:
  - Val F1 ≥ 0.75
  - Val AUC ≥ 0.80

현재 적용 가능 모델: SigLIP, CLIP, DINOv2 ✅

사용 사례:
  - 장기 안전 트렌드 분석
  - 위험 요소 빈도 통계
  - 안전 교육 자료 생성
```

### Ensemble 효과

**단일 모델 vs Ensemble 비교 (예상)**:

| 구성 | Val F1 | Val AUC | 신뢰도 |
|------|--------|---------|--------|
| SigLIP 단독 | 0.9566 | 0.9917 | 높음 |
| CLIP 단독 | 0.8776 | 0.9484 | 중간 |
| DINOv2 단독 | 0.8408 | 0.9229 | 중간 |
| **Ensemble (3개)** | **0.96~0.97** | **0.995+** | **매우 높음** |

**Ensemble 장점**:
- 각 모델의 강점 결합
- 분포 변화(distribution shift)에 강건
- 확신도 낮은 경우 다수결로 보완

- 추락 위험 지역이지만 난간이 가려짐

개선 방법:
- 다중 각도 카메라 설치
- 시간 순서 분석 (비디오 스트림)

```

- 역광으로 인한 실루엣

- 강한 그림자

개선 방법:
- 적외선 카메라 추가
- 조명 보강
- 이미지 전처리 강화

3. **새로운 위험 유형 (18%)**:
```
예시:
- 학습 데이터에 없는 신규 장비

- 지속적 데이터 수집

- Domain Adaptation

4. **미세한 위험 (11%)**:
```

- 미끄러운 바닥 (시각적 구분 어려움)

개선 방법:

- 깊이 정보 부족
- 거리 판단 어려움 (충돌 위험 평가 시)
- 작은 물체 탐지 제한 (원거리)

예시:
- 10m 떨어진 중장비와 작업자
  → 위험한지 안전한지 판단 어려움
- 안전벨트 연결 고리 확인
  → 해상도 부족 시 불가능

해결 방안:
- Stereo 카메라 (깊이 추정)
- Lidar 센서 추가
- 고해상도 카메라 (4K+)
- Zoom-in 기능
```

**2. 시간적 맥락 부재**:
```
현재: 단일 이미지 분석
→ 이전/이후 상황 모름

문제 예시:
- 작업자가 잠깐 안전모를 벗었는지
- 지속적으로 벗고 있는지 구분 불가
- 위험 구역 진입 vs 진출 구분 어려움

해결 방안:
- 비디오 분석 (시간 시퀀스)
- LSTM/Transformer 기반 모델
- 작업자 추적 (Object Tracking)
```

**3. 환경 의존성**:
```
학습 데이터 분포:
- 한국 건설 현장 위주
- 주간 작업 중심
- 맑은 날씨 다수

일반화 한계:
- 해외 현장 (다른 안전 기준)
- 극한 환경 (사막, 극지)
- 특수 산업 (조선, 석유화학)

Domain Shift:
- F1 저하: 5~15%p
- 재학습 필요

해결 방안:
- 다양한 환경 데이터 수집
- Domain Adaptation
- Transfer Learning
```

### 윤리적/법적 고려사항

**1. 개인정보 보호**:
```
이슈:
- CCTV는 작업자 얼굴 포함
- GDPR, 개인정보보호법 준수 필요

SafetyKnob의 접근:
- 얼굴 정보 미사용 (자동 블러 처리 옵션)
- 안전 판단에 신원 불필요
- 익명화된 통계만 저장

권장사항:
- 작업자 사전 고지 및 동의
- 영상 보관 기간 제한 (7~30일)
- 접근 권한 엄격 관리
```

**2. 책임 소재**:
```
AI 경보를 무시한 사고 발생 시:
- 관리자 책임? AI 책임?
- 법적 공백 존재

현실적 해결:
- AI는 "보조 도구"로 명시
- 최종 판단/조치는 인간
- AI 권고 이력 문서화
- 보험 및 면책 조항 검토

판례 부재:
- 국내외 아직 선례 없음
- 법적 프레임워크 발전 중
```

**3. 과신/의존 위험**:
```
Automation Bias:
- AI를 맹신하여 인간 판단력 저하
- "AI가 괜찮다고 했으니까"

실제 사례 (가상):
"관리자가 AI 경보가 없어서 안전하다고 판단,
 실제로는 카메라 사각지대에 위험 존재,
 사고 발생"

예방책:
- 정기 인간 검수 (랜덤 샘플링)
- AI 한계 교육
- 의심스러울 때는 항상 직접 확인
```

### 사용 시 주의사항

**1. 임계값 설정**:
```
잘못된 예:
- 모든 현장에 획일적으로 0.5 적용

올바른 예:
- 고위험 (고소작업): 0.3~0.4
- 중위험 (일반 건설): 0.5
- 저위험 (정리정돈): 0.6~0.7

주의:
- 초기 2주는 0.3으로 시작
- 오경보율 관찰 후 조정
- 현장별 최적값 찾기
```

**2. 정기 재학습**:
```
필요성:
- 새로운 장비 도입
- 작업 방식 변화
- 계절 변화 (조명 조건)

권장 주기:
- 분기별: 데이터 수집
- 반기별: 모델 업데이트
- 연 1회: 대규모 재학습

주의:
- 구버전 모델 백업
- A/B 테스트 후 배포
```

**3. 카메라 유지보수**:
```
흔한 문제:
- 렌즈 오염 (먼지, 빗방울)
- 각도 변화 (바람, 진동)
- 조도 센서 고장

영향:
- F1 저하: 10~30%p
- 오경보 급증

예방:
- 주 1회: 렌즈 청소
- 월 1회: 각도/초점 확인
- 분기 1회: 센서 캘리브레이션
```

### 보완 권장사항

**단기 (즉시)**:
```
1. 인간-AI 협업 프로토콜 수립
   - 확신도별 대응 매뉴얼
   - Escalation 절차

2. 오경보 피드백 루프
   - 관리자가 FP 표시
   - 다음 학습에 반영

3. 다중 카메라 배치
   - 사각지대 최소화
   - 3D 복원 가능
```

**중기 (3~6개월)**:
```
1. 비디오 분석 확장
   - 시간 순서 고려
   - 작업자 동선 추적

2. 센서 융합
   - GPS (위치)
   - 가속도계 (움직임)
   - 온도/가스 센서

3. Explainable AI
   - 왜 위험하다고 판단했는지 설명
   - Attention map 시각화
```

**장기 (1년+)**:
```
1. Edge Computing
   - 현장 즉시 처리
   - 네트워크 지연 제거
   - 프라이버시 강화

2. 예측적 안전 관리
   - 사고 발생 예측
   - 위험 패턴 분석
   - 선제적 대응

3. 디지털 트윈 통합
   - 3D 모델과 연계
   - 가상 시뮬레이션
   - 최적 안전 계획 수립
```
