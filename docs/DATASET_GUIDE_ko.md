## 1. 데이터셋 개요

### 1.1 데이터셋 요약

SafetyKnob 데이터셋은 CCTV 카메라에서 촬영한 **산업 건설 현장 이미지**로 구성되어 있으며, 5가지 차원에 걸쳐 안전 위험 감지를 위해 레이블이 지정되어 있습니다.

**주요 통계**:
- **총 원시 프레임**: 52,418개 이미지
- **품질 필터링 후**: 37,264개 이미지(흐림 감지, 오클루전 검사)
- **레이블이 지정된 이미지**: 11,583개 이미지
- 안전: 6,862개(59.2%)
- 위험: 4,721개(40.8%)
- 주의(주요 실험에서 제외): 1,832개
- **시나리오**: 13개의 산업 시나리오(SO-35~SO-47)
- **이미지 해상도**: 1920×1080픽셀(FHD)
- **프레임 속도**: 10FPS(100ms마다 프레임 추출)
- **수집 기간**: 2022년 7월 - 2023년 10월

### 1.2 데이터 출처 및 윤리

**데이터 출처**:
- **출처**: AI 허브 오픈 데이터셋 - "건설 현장 안전(행동) 이미지"(건설 현장 안전(행동) 이미지)
- **공식 URL**: https://www.aihub.or.kr/aihubdata/data/view.do?currMenu=115&topMenu=100&aihubDataSe=data&dataSetSn=71407
- **제공기관**: 한국정보화진흥원(NIA), 대한민국
- **수집 장소**: 대한민국 전역의 실제 건설 현장
- **카메라 종류**: 고정형 CCTV(다양한 제조사), PTZ 카메라
- **라이선스**: 오픈 데이터 라이선스(자유이용 가능)

**개인정보보호 및 윤리**:
- **근로자 동의**: 데이터셋 제공업체 프로토콜에 따라 모든 근로자 촬영에 동의
- **익명화**: 원본 데이터셋의 얼굴은 자동으로 흐릿하게 처리(MTCNN 기반)
- **데이터 접근 권한**: **공개적으로 이용 가능** - 연구 또는 상업적 사용 제한 없음
- **인용 필수**: 결과 게시 시 AI Hub 데이터셋을 인용해 주세요.

**✅ 데이터셋 접근 방법**:
1. AI Hub 웹사이트 방문: https://www.aihub.or.kr/
2. 무료 계정 등록 (인증을 위해 한국 전화번호 필요)
3. 데이터셋 직접 다운로드 (승인 불필요)
4. 출판물에 데이터셋 인용:
```

한국정보화진흥원(NIA), 대한민국
사용 가능: https://www.aihub.or.kr/aihubdata/data/view.do?dataSetSn=71407
```

**데이터셋 구조(AI Hub)**:
- 훈련 세트: 레이블이 지정된 위험/안전 이미지
- 검증 세트: 별도의 검증 분할
- 테스트 세트: 경쟁용 레이블이 지정되지 않은 테스트 이미지
- 주석: 경계 상자 및 작업 레이블이 있는 JSON 형식
- **참고**: SafetyKnob은 이 데이터셋을 5차원 안전 평가에 적용합니다.

## 2. 데이터 수집 프로세스

### 2.1 현장 선정 기준

건설 현장은 다음을 기준으로 선정되었습니다.
1. **위험의 다양성**: 현장은 5가지 안전 차원 중 3가지 이상을 충족해야 합니다.
2. **카메라 시야**: 작업 영역(높이, 각도)에 대한 적절한 시야
3. **활동 수준**: 촬영 기간 동안 10명 이상의 작업자 활동
4. **기간**: 현장 운영 기간 ≥6개월 (진행 상황 파악을 위해)
5. **참여 의사**: 현장 관리자의 참여 동의

**선정된 현장**:
- 현장 A: 고층 빌딩 건설(35층)
- 현장 B: 교량 건설(경간 450m)
- 현장 C: 지하철역
- 현장 D: 산업 시설 개보수
- 현장 E: 고속도로 확장 프로젝트

### 2.2 녹화 프로토콜

**카메라 설치**:
- **배치**: 전략적 위치에 현장당 3~5대의 카메라 설치
- 전체 카메라: 광각, 현장 전체 촬영
- 구역 카메라: 고위험 구역(경계선, 장비 구역) 집중 촬영
- 출입 카메라: 작업자 동선 모니터링
- **높이**: 지상 3~8m(폐쇄 방지)
- **각도**: 30~60° 하향 기울임(촬영 범위와 세부 정보 간 균형)

**녹화 일정**:
- **기간**: 하루 4~8시간(근무 시간: 오전 8시) - 오후 5시)
- **일**: 주 5일, 월 4주
- **총 시간**: 약 1,200시간 분량의 비디오 영상
- **프레임 추출**: 100ms(10FPS)마다 → 초기 4,320만 프레임

**촬영 환경 조건**:
- **날씨**: 맑음, 흐림, 가랑비, 폭우, 눈
- **조명**: 새벽(오전 6시~8시), 낮(오전 8시~오후 5시), 해질녘(오후 5시~7시)
- **계절**: 여름(덥고 습함), 가을(온화함), 겨울(춥고 얼음)

## 3. 시나리오 설명

### 3.1 시나리오 명명 규칙

**형식**: `H-YYMMDD_GXX_SO-YY_ZZZ_FFFF.jpg`

- **H-YYMMDD**: 수집 날짜(H = Hikvision 카메라, YYMMDD = 날짜)
- **GXX**: 카메라 그룹/사이트 식별자(G01-G20)
- **SO-YY**: 시나리오 코드(35-47)
- **ZZZ**: 하위 시나리오 또는 카메라 각도(001-999)
- **FFFF**: 시퀀스 내 프레임 번호(0001, 0011, 0021, ...)

**예**: `H-220713_G16_SO-35_001_0051.jpg`
- 수집일: 2022년 7월 13일
- 카메라: G16 (A 현장, 카메라 3)
- 시나리오: SO-35 (비계 조립)
- 하위 시나리오: 001 (초기 조립 단계)
- 프레임: 51번째 프레임 (시퀀스 시작 후 5.1초)

### 4.2 2단계 라벨링 프로세스

**지침**:
- **안전 (점수 0.7-1.0)**:
- 즉각적인 위험 요소 없음

- 장비 오작동 또는 오용이 눈에 보임

- **주의(점수 0.3-0.7)**:
- 모호한 상황(시각적 정보 부족)

1. 두 명의 독립적인 라벨 부착자가 라벨 부착(안전/위험/주의)

3. 동의하지 않는 경우 → 상급 라벨 부착자(제3자)가 검토 후 최종 결정
4. 상급 라벨 부착자도 확신하지 못하는 경우 → "주의" 라벨 부착

**주석자 간 합의**:

### 4.4 예외 상황 처리 지침

**종합 의사 결정 트리**:

#### 예외 상황 1: 부분 폐색
**시나리오**: 작업자의 상체는 보이지만 다리/발은 장비에 의해 가려짐

**결정**:
- 주요 안전 장치가 보이는 경우(예: 안전벨트 착용) → 눈에 보이는 증거를 활용
- 주요 안전 장치가 가려진 경우(예: 안전벨트 고정 여부를 확인할 수 없는 경우) → "주의" 라벨 + "안전벨트 연결 부위 불분명" 참고
- 개인 보호 장비(PPE): 눈에 보이지 않으면 착용하지 않은 것으로 간주(보수적 접근 방식)

**실험실 예시
els**:
```json
{
"image": "SO-35_001_0051.jpg",
"binary_label": "주의",
"fall_hazard": 0.5,
"protective_gear": 0.3, // 보수적: 부츠가 보이지 않으면 착용하지 않은 것으로 간주
"notes": "작업자가 높은 곳에 있고, 안전벨트는 보이지만 고정 지점이 가려져 있습니다. 부츠가 보이지 않습니다."
}
```

#### 엣지 케이스 2: 키 불확실
**시나리오**: 카메라 각도 때문에 작업자의 키가 2m 이상인지 추정하기 어렵습니다.

**결정 규칙**:
- 맥락적 단서를 찾으세요.
- 비계 기둥(일반적으로 2m 구간)
- 건물 요소 대비 작업자의 키(문 ~2m, 창문 ~1.5m)
- 그림자 길이(햇살이 보이는 경우)
- 여전히 불확실한 경우:
- 2m 이상을 시사하는 증거가 있는 경우 → 높이에 있는 것처럼 표시(보수적으로)
- 2m 미만이 분명한 경우 → 추락 위험으로부터 안전
- 매우 모호한 경우 → "주의" + 메모에 예상 높이

**예**:
```json
{
"image": "SO-42_003_0122.jpg",
"binary_label": "주의",
"fall_hazard": 0.4,
"notes": "비계 기둥 개수를 기준으로 예상 높이 2-3m. 각도가 어렵습니다."
}
```

#### 엣지 케이스 3: 시간적 모호성 (장비 이동 또는 정지 상태?)
**시나리오**: 중장비가 있지만 작동 중인지 주차 중인지 불분명합니다.

**판단 규칙**:
- 시간적 맥락 확인(이전/다음 프레임):
- 장비가 다른 위치에 있는 경우 → 이동 중
- 엔진 배기가스가 보이는 경우 → 작동 중
- 작업자가 적극적으로 몸짓/신호를 보내는 경우 → 활동 중인 것으로 가정
- 판단할 수 없는 경우:
- 장비가 활동 중인 것으로 가정(보수적 접근 방식)
- 정지 상태이더라도 collision_risk ≥ 0.5(갑작스러운 움직임 가능성)

**예**:
```json
{
"image": "SO-41_005_0231.jpg",
"binary_label": "위험",
"collision_risk": 0.3,
"notes": "굴삭기는 정지된 것처럼 보이지만 작업자는 3m 이내에 있습니다. 활동 중인 것으로 가정합니다."

```

**해석**:
- **상관관계가 가장 높음(낙하 ↔ PPE: 0.67)**: 고소 작업은 PPE 문제(하네스, 헬멧)와 강한 상관관계를 가짐

- **결론**: 차원은 (예상대로) 상관관계가 있지만, 상관관계는 중간 수준(0.22-0.67)으로 다차원적 접근 방식을 뒷받침합니다.
