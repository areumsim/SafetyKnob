## 6. 5차원 라벨 자동 생성 전략

### 현재 라벨 구조
```json
{
  "train/H-220927_A35_N-07_005_0061.jpg": {
    "overall_safety": 0,
    "class": "danger"
  }
}
```

### 목표 5차원 라벨 구조
```json
{
  "train/H-220927_A35_N-07_005_0061.jpg": {
    "overall_safety": 0,
    "class": "danger",
    "dimensions": {
      "fall_hazard": 0.0,        # A35 → Fall
      "collision_risk": 0.9,     # Not B
      "equipment_hazard": 0.9,   # Not C
      "environmental_risk": 0.9, # Not D
      "protective_gear": 0.9     # Not E
    }
  }
}
```

### 라벨링 규칙

**Rule 1: 사고 코드 추출**
```python
import re
filename = "H-220927_A35_N-07_005_0061.jpg"
match = re.search(r'_([A-F]\d{2})_', filename)
accident_code = match.group(1)  # "A35"
category = accident_code[0]     # "A"
```

**Rule 2: 카테고리 매핑**
```python
category_mapping = {
    'A': 'fall_hazard',
    'B': 'collision_risk',
    'C': 'equipment_hazard',
    'D': 'environmental_risk',
    'E': 'protective_gear'
}
```

**Rule 3: 차원별 라벨 생성**
```python
# 초기화: 모든 차원 0.9 (not applicable)
dimensions = {dim: 0.9 for dim in category_mapping.values()}

# 해당 차원만 danger/safe 라벨 적용
target_dimension = category_mapping[category]
dimensions[target_dimension] = 1.0 if overall_safety == 1 else 0.0
```

## 7. Distribution Shift 실험 설계

### Date-based Split 전략

**Train Set**: 2022년 6-9월
```
날짜 범위: 220607 ~ 220930
기간: 116일
예상 이미지: ~10,000개 (70%)
특징: 여름~가을, 밝은 조명, 반팔 작업복
```

**Validation Set**: 2022년 10월
```
날짜 범위: 221001 ~ 221031
기간: 31일
예상 이미지: ~2,000개 (14%)
특징: 가을, 중간 조명, 긴팔 작업복
```

**Test Set**: 2022년 11월
```
날짜 범위: 221101 ~ 221130
기간: 30일
예상 이미지: ~2,000개 (16%)
특징: 가을~초겨울, 어두운 조명, 외투
```

### 예상 Distribution Shift 효과

**1. 조명 변화**
- 여름 (Train): 높은 자연광, 밝은 환경
- 겨울 (Test): 낮은 자연광, 인공조명 의존

**2. 작업자 복장**
- 여름 (Train): 반팔, 얇은 조끼
- 겨울 (Test): 긴팔, 두꺼운 외투

**3. 환경 배경**
- 여름 (Train): 녹색 식물, 맑은 날씨
- 겨울 (Test): 갈색 식물, 흐린 날씨

**4. 예상 성능 하락**
```
Baseline (Random split): F1 0.9573
Date split (expected):   F1 0.92-0.95 (-3 to -8%)
```

### Location-based Split (선택)

**Option 2**: 현장별 분리
```
Train: A*, B*, C* locations (8,000~9,000개)
Test:  D*, E*, F* locations (3,000~4,000개)
```

**장점**: 더 강한 distribution shift
**단점**: 데이터 불균형 가능성
